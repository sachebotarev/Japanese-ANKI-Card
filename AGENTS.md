# AGENTS

Короткая инструкция для ИИ-агента, работающего в этом репозитории.

## Что делать

- Работать с проектом как с пайплайном `Слова/ -> Карточки/ -> Картинки/ -> Произношение/ -> Anki`.
- Использовать русский язык для переводов, пояснений и заметок.
- Считать `docs/` основным источником подробных правил.
- Перед массовыми изменениями или синхронизацией с Anki сверять актуальные поля заметки через инструменты Anki.

## Куда смотреть

- Формат карточек и правила полей: [docs/cards-spec.md](docs/cards-spec.md)
- Синхронизация с Anki: [docs/anki-sync.md](docs/anki-sync.md)
- Озвучка: [docs/audio.md](docs/audio.md)
- Обработка файлов в `Слова/`: [docs/word-lists.md](docs/word-lists.md)

## Оперативные правила

- Перед обработкой файлов в `Слова/` нужно удалить пустые строки, убрать дубли и отсортировать строки.
- Одна карточка хранится в одном JSON-файле.
- Тема должна совпадать между `Слова/`, `Карточки/`, `Картинки/` и `Произношение/`.
- Если пользователь дал слово без указания темы или файла, тему нужно определить самостоятельно. Сначала выбирай наиболее подходящую уже существующую тему, а новую создавай только если разумно обойтись без неё нельзя.
- Если добавляется существительное, его нужно классифицировать по теме применения, а не по части речи как таковой. Сначала ищи подходящую уже существующую тематическую категорию вроде `Профессии`, `Части тела`, бытовой или предметной группы.
- Тему `Существительные` используй только как запасной вариант, когда для существительного нет более точной и устойчивой тематической категории среди уже существующих тем.
- Если для существительного действительно нужна новая тема, её можно создать только когда слово и ожидаемые соседние слова образуют осмысленную устойчивую группу, а существующие темы подходят заметно хуже.
- Не нужно плодить новые темы ради одного частного существительного, если его можно без натяжки отнести к уже существующей теме или временно оставить в `Существительные`.
- Если пользователь дал слово без указания темы или файла, агент обязан не только выбрать тему, но и фактически отнести слово в соответствующий файл `Слова/<тема>` и в соответствующую папку `Карточки/<тема>/`, если для выбранного файла нет отдельного явного запрета.
- Нельзя считать это правило выполненным, если агент создал только JSON-карточку, но не обновил соответствующий файл в `Слова/`, когда обновление этого файла разрешено.
- `Слова/Кандзи` не нужно пополнять автоматически. Этот файл пользователь ведёт вручную, поэтому агент не должен добавлять туда строки без прямой просьбы.
- Для `Слова/Кандзи` прямой просьбой считается только явное указание пользователя добавить слово именно в `Слова/Кандзи` или явно разрешить изменить этот файл. Общие формулировки вроде `в нужный файл`, `добавь слово` или `полный пайплайн` таким разрешением не считаются.
- Если выбранная тема подпадает под явное исключение вроде `Слова/Кандзи`, агент должен явно сообщить пользователю, что слово сознательно не добавлено в `Слова/<тема>` из-за этого правила, а не молча пропускать шаг.
- Если в JSON поле `Картинка` хранит текстовый промпт и есть техническая возможность генерации, картинку нужно создавать сразу, без дополнительных вопросов пользователю, сохранить в `Картинки/<тема>/`, затем заполнить ей поле `Картинка` в Anki.
- Для генерации иллюстраций карточек нужно использовать нормальную raster AI-генерацию, а не самодельные плейсхолдеры, примитивные программные рисунки, SVG-заглушки или схематичные иконки вместо полноценной сцены.
- Если доступен встроенный инструмент генерации изображений, для карточек его нужно считать основным способом создания иллюстраций.
- Если карточки нет в Anki, её нужно создать из JSON.
- Если карточка уже есть, обновлять её нужно по правилам из `docs/anki-sync.md`.
- После создания новой карточки и связанных с ней файлов агент должен сделать git-коммит и залить изменения в удалённый репозиторий, если пользователь явно не запретил это отдельным указанием.
- Если в JSON поле `Произношение` пустое, озвучку нужно делать по правилам из `docs/audio.md`, не затирая уже имеющееся аудио в Anki.
- Если в локальном `AivisSpeech` доступен голос `Anneli-nsfw`, для озвучки по умолчанию нужно использовать его. Если он недоступен, допустим fallback `まお`.
- Для озвучки через `AivisSpeech` по умолчанию нужно использовать скорость `90%`.
- Если пользователь пишет короткий запрос вроде `Добавь слово 口座` или `Создай карточку для слова 口座`, это нужно трактовать как запрос на полный пайплайн, если пользователь явно не ограничил объём работы.
- Для короткого запроса полный пайплайн по умолчанию включает и обновление `Слова/<тема>`, если для выбранной темы это не запрещено отдельным правилом.
- Исключение `Слова/Кандзи` имеет приоритет над общим правилом полного пайплайна.
