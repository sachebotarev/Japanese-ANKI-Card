# Правило генерации промтов для ComfyUI

## Цель

Использовать это правило, чтобы по любой русской фразе быстро получать стабильный промт для `Nova Anime XL` без лишнего текста, артефактов и случайных персонажей.

## Базовый шаблон (Positive Prompt)

```text
masterpiece, best quality, anime illustration, [композиция/ракурс], [строгое количество людей], [место действия], [кто что делает], [ключевой объект сцены как физический предмет], [эмоция/атмосфера], [свет], [фоновые детали], natural poses, detailed hands, cinematic storytelling composition
```

## Обязательные правила заполнения

- Пиши короткими визуальными блоками через запятую.
- Сначала фиксируй стиль и композицию, потом действие, потом атмосферу.
- Если важно число людей, обязательно добавляй:
  - `exactly two people, two people only, no other people`
- Если важен предмет (например, словарь), описывай его как физический объект:
  - `physical book object, plain book cover without any letters`
- Чтобы не появлялись надписи на объектах, добавляй запрет текста и в positive, и в negative.
- Избегай длинных абстрактных фраз; каждая часть должна быть визуально проверяемой.

## Базовый Negative Prompt

```text
third person, extra person, crowd, background people, text, letters, typography, words, subtitles, watermark, logo, signature, kanji on objects, worst quality, low quality, blurry, bad anatomy, bad hands, extra fingers, deformed face
```

## Настройки по умолчанию (Nova Anime XL)

- `Sampler`: `DPM++ 2M Karras`
- `Steps`: `32-40`
- `CFG`: `6.0-6.8`
- `Size`: `768x1024` или `832x1216`

## Алгоритм: как превратить фразу в промт

1. Выдели 5 смысловых блоков фразы:
   - кто участвует;
   - где происходит;
   - какое действие;
   - какой ключевой предмет;
   - какое настроение.
2. Сконвертируй блоки в визуальные формулировки на английском.
3. Добавь жёсткие ограничения (число людей, отсутствие текста на предметах).
4. Собери финальный positive prompt по базовому шаблону.
5. Добавь базовый negative prompt без изменений.
6. При необходимости сделай 3-5 разных `seed`, не меняя логику промта.

## Мини-чек перед генерацией

- Есть ли явное указание количества людей?
- Есть ли явное описание ключевого предмета?
- Есть ли запрет текста/букв?
- Есть ли в negative ограничения на лишних персонажей?
- Нет ли в prompt слов, провоцирующих модель рисовать надписи?
