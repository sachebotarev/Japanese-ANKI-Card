# Синхронизация с Anki

## Общие правила

- Для работы с карточками в Anki используется MCP-инструмент `anki`.
- Основная колода: `Японские слова`.
- Основной тип заметки: `Японские слова`.
- Перед созданием или массовым обновлением карточек нужно сверять актуальные поля заметки через инструменты Anki.

## Рабочий процесс

1. Найти в `Слова/` слова, для которых ещё нет JSON-карточек.
   Если пользователь дал новое слово без темы, сначала отнести его в подходящий файл `Слова/<тема>` и нормализовать этот файл, кроме случаев с отдельным явным запретом вроде `Слова/Кандзи`.
   Для `Слова/Кандзи` запрет снимается только явной просьбой пользователя изменить именно этот файл.
2. Создать для них JSON-файлы в `Карточки/<тема>/`.
3. Если в JSON поле `Картинка` содержит текстовый промпт и есть возможность генерации, нужно сразу создать или обновить изображение в `Картинки/<тема>/`, без дополнительных вопросов пользователю, а затем загрузить его в Anki. Промпт и сама картинка должны опираться на третий пример.
4. Если в JSON поле `Произношение` пустое, при необходимости сгенерировать озвучку в `Произношение/<тема>/` для слова и всех трёх примеров.
5. Проверить формат и соответствие структуры инструментами проекта.
6. Создать или обновить заметки в Anki через MCP `anki`.

Для перехода `готовый файл картинки -> collection.media Anki -> поле Картинка в JSON` можно использовать `scripts/import_card_image.py`.

Если шаг обновления `Слова/<тема>` пропущен из-за отдельного правила-исключения, это нужно явно сообщать пользователю в результате работы.

## Правила обновления карточек

- Если карточки в Anki нет, её нужно создать и заполнить все поля значениями из JSON.
- Теги заметки в Anki нужно брать из поля `Теги` в JSON.
- Если карточка уже существует, поля нужно обновить из JSON, кроме специальных случаев ниже.
- Если поле **Пример** в Anki не совпадает с полем `Пример` в JSON, нужно перезаписать в Anki поля **Картинка** и **Произношение** значениями, актуальными для JSON.
- Если поле **Пример** в Anki совпадает с полем `Пример` в JSON, заполненное поле **Произношение** можно не менять.
- Если в Anki поле **Картинка** уже содержит изображение и поле **Пример** совпадает с полем `Пример` в JSON, картинку менять не нужно.
- Если в Anki поле **Картинка** пустое или содержит только текстовый промпт, а генерация доступна, нужно сразу создать локальную картинку без дополнительных вопросов пользователю и заменить содержимое поля **Картинка** в Anki на саму картинку.
- Если поле **Пример** отличается, сначала в JSON поле **Картинка** нужно записать новый промпт для генерации по третьему примеру, затем при наличии возможности создать по нему новую локальную картинку и обновить поле **Картинка** в Anki самой картинкой; поле **Произношение** в Anki также нужно обновить в соответствии с текущим JSON.
- Если изменился любой из трёх примеров в поле `Пример`, это считается несовпадением примеров между Anki и JSON и требует обновления полей **Картинка** и **Произношение** в Anki по актуальному JSON.
- Если генерация недоступна, в JSON и Anki временно допускается оставить текстовый промпт, но это считается промежуточным состоянием, а не финальным результатом.

## Обратная синхронизация в JSON

После обновления карточки в Anki нужно синхронизировать JSON-файл в `Карточки/`:

- добавить поля, которые есть в Anki, но отсутствуют в JSON;
- обновить в JSON значения полей, которые были изменены в Anki;
- учитывать, что после такой синхронизации поле `Картинка` в JSON обычно уже содержит финальное HTML или ссылку из Anki, а не исходный промпт.
