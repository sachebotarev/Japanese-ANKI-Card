# Заметки в каталоге «Записи»

Заметки в формате Markdown для чтения в Obsidian (или другом редакторе) делаются **по JSON-карточке** из `Карточки/<тема>/`. Источник правды по смыслу полей — [cards-spec.md](cards-spec.md).

**Для агента:** при **создании новой** карточки в репозитории заметку в `Записи/` нужно сформировать по этому документу в том же проходе, что и остальной пайплайн карточки (см. [AGENTS.md](../AGENTS.md)). Если заметку собрать нельзя — файл не создавать и сообщить пользователю причину (см. правило в AGENTS.md).

Хранилище Obsidian должно быть открыто **корнем этого репозитория**, чтобы ссылки вида `![[Картинки/...]]` и `![[Произношение/...]]` работали без правок путей.

## Размещение файлов

- Каталог: **`Записи/<тема>/`**, где `<тема>` совпадает с подпапкой в `Карточки/` (например, `Глаголы`).
- Имя файла: **как у JSON-карточки**, без расширения — например `差す.md` для `Карточки/Глаголы/差す.json`.

## YAML в начале файла

Заполнять по данным из JSON-карточки:

- **`карточка`** — относительный путь к JSON, например `Карточки/Глаголы/差す.json`.
- **`колода_anki`** — основная колода проекта (`Японские слова`).
- **`тема`** — та же строка, что подкаталог (`Глаголы` и т.д.).
- **`tags`** — массив из поля `Теги` в JSON; значения с символами вроде `・` удобно брать в кавычки. Правила тегов, в том числе привязка к уроку Minna no Nihongo, задаются в [cards-spec.md](cards-spec.md) (раздел «Теги»). Ключ называется по-английски (`tags`), потому что ванильный Quartz читает frontmatter именно по этому имени; индекс и фильтрация на сайте берутся из него (см. [quartz-site.md](quartz-site.md)).
- **`показать_фуригану`** — значение поля `Показать фуригану` из JSON (`Y` / `N`).

## Тело заметки: что выводить

Порядок блоков **фиксированный**.

1. **Заголовок первого уровня** — поле `Слово` (кандзи-запись слова).
2. **Чтение** — одна строка из поля `Чтение` (хирагана/катакана), без подписи «Чтение:».
3. **Перевод** — одна или несколько строк из поля `Перевод`, без подписи «Перевод:».
4. **Озвучка слова** — одна вставка Obsidian: `![[Произношение/<тема>/<Слово>_слово.mp3]]` сразу после перевода. Отдельного раздела «Произношение» **не делать** — аудио слова относится к блоку со словом.
5. **Картинка** — одна строка: `![[Картинки/<тема>/<Слово>.png]]` (или актуальное расширение файла в проекте). Без подписей, пояснений про Anki и имени файла в media.
6. **`## Примеры`** — ровно три подраздела **`### Пример 1`**, **`### Пример 2`**, **`### Пример 3`** в том же порядке, что и три фразы по разбиению `<br>` в полях `Пример` и `Пример перевод`. Строки из поля `Пример без слова` (японский с пропусками `_____` для карточки Anki) **в заметку и на сайт Quartz не выводить** — только японский полный пример и русский перевод.

В каждом **`### Пример N`**:

- сначала озвучка этого примера: `![[Произношение/<тема>/<Слово>_пример_N.mp3]]` (N = 1, 2, 3);
- затем **два** абзаца подряд (пустая строка между абзацами):
  1. японская фраза из `Пример` для этой позиции; изучаемое слово выделить жирным `**...**` (вместо HTML `<span class='study-word'>` из JSON);
  2. соответствующий фрагмент из `Пример перевод` (русский текст).

Разбиение на три примера — по разделителю `<br>` в JSON: первая часть → Пример 1, вторая → Пример 2, третья → Пример 3. Поле `Пример без слова` в карточке JSON по-прежнему нужно соблюдать по [cards-spec.md](cards-spec.md), но Markdown-заметка его не повторяет.

7. **`## Заметки`** — текст поля `Заметки`.
8. **`## Кандзи-разбор`** — текст поля `Кандзи-разбор`.

## Что в заметку не включать

Чтобы заметка оставалась читаемой и без лишней техники:

- поле **Подсказка**;
- отдельный заголовок **«Произношение»** и сводный список всех аудио в одном месте (озвучка слова — только в блоке со словом; озвучка примеров — только внутри соответствующего `### Пример N`);
- пояснения про поле «Показать фуригану» в теле заметки (достаточно значения в YAML);
- строки **`Пример без слова`** (пропуски `_____`) — они для вида карточки в Anki, на сайте и в чтении заметки не нужны;
- дамп JSON, HTML из карточки, имена файлов Anki media (`japanese_words_…`), строки `[sound:…]` — если не нужны лично вам для отладки;
- таблицы и дублирование одних и тех же данных в разных формах без необходимости.

При изменении примеров или аудио в JSON сначала обновляют карточку и файлы в `Произношение/`, затем при необходимости подправляют заметку в `Записи/`.

Массовая генерация по всем карточкам: скрипт [generate_zapisi_from_cards.py](../scripts/generate_zapisi_from_cards.py), отчёт о пропусках — [zapisi-generation-report.md](zapisi-generation-report.md).
