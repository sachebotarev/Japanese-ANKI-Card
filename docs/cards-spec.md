# Спецификация карточек

## Общие принципы

- Все переводы, пояснения и заметки в карточках должны быть на русском языке.
- Основная колода Anki: `Японские слова`.
- Основной тип заметки Anki: `Японские слова`.
- Перед созданием или массовым обновлением карточек нужно сверять актуальные поля заметки через доступные инструменты Anki.
- Одна карточка хранится в одном JSON-файле.

## Обязательные поля JSON

- **Слово** — словарная форма, по возможности в кандзи.
- **Перевод** — общепринятый русский перевод, предпочтительно по авторитетному словарю.
- **Чтение** — хирагана или катакана.
- **Подсказка** — мнемоника без прямого раскрытия целевого слова и без однокоренных подсказок.
- **Пример** — одно текстовое поле с ровно тремя японскими предложениями, разделёнными тегом `<br>`. Во всех трёх примерах изучаемое слово нужно оборачивать в `<span class='study-word'>...</span>`. Примеры идут в фиксированном порядке: первый — простое предложение начального уровня, второй — предложение средней сложности с грамматикой не выше N5, третий — сложное комплексное предложение с несколькими грамматическими конструкциями, но не выше N4.
- **Пример без слова** — те же три предложения в одном поле через тег `<br>`, где целевое слово в каждом примере заменено на `_____`. Порядок примеров должен строго совпадать с полем **Пример**.
- **Пример перевод** — три русских перевода в одном поле через тег `<br>`, по одному переводу на каждый пример поля **Пример** и в том же порядке.
- **Картинка** — на этапе подготовки карточки это текстовый промпт для генерации изображения 512×512 по смыслу третьего примера из поля **Пример перевод**. После загрузки изображения в Anki это поле может быть синхронизировано обратно как HTML или ссылка на уже используемую картинку.
- **Произношение** — при первичном создании карточки не заполняется. После генерации аудио обычно содержит 4 ссылки: `[sound:...][sound:...][sound:...][sound:...]` для слова и трёх примеров.
- **Заметки** — до двух предложений для запоминания и понимания. Для глаголов сюда нужно добавлять информацию о типичных частицах и модели управления.
- **Кандзи-разбор** — разбор кандзи в слове.
- **Показать фуригану** — `N`, если в слове есть хотя бы один кандзи из объёма N4/N5, иначе `Y`.
- **Теги** — массив строк с тегами проекта. Используются как теги заметки в Anki и **дословно** копируются в YAML ключ **`tags`** заметки в `Записи/` при сборке по [zapisi-spec.md](zapisi-spec.md). На сайте Quartz те же строки становятся **кликабельными тегами** над текстом страницы и попадают в индекс **`/tags/`**, так что по тегу урока Minna (формат см. ниже в разделе «Теги») можно отобрать все карточки этого урока.

## Теги

- Поле `Теги` в JSON должно содержать все теги, которые нужно применить к заметке в Anki.
- Для глаголов нужно добавлять тег `動詞`.
- Для существительных нужно добавлять тег `名詞`.
- Для слова нужно добавлять тег уровня JLPT: `N5`, `N4`, `N3`, `N2` или `N1`.
- Для глаголов нужно добавлять тег со спряжением.
- Для карточки нужно добавлять тег, совпадающий с поддиректорией в `Карточки/`, где лежит JSON-файл.
- **Учебник «みんなの日本語» (Minna no Nihongo):** если изучаемое слово **относится к лексике** этого учебника (том 初級 I или 初級 II), в `Теги` обязательно добавляют **один** тег с **номером урока（第…課）** в оглавлении **того тома**, где это слово вводится как учебная единица (в терминах учебника это соответствует «главе» в смысле раздела для прохождения — **урок**, а не отдельная книжная 章). Формат строки тега:
  - `みんな初級I-第NN課` — урок `NN` тома 初級 I (`NN` с ведущим нулём: `01` … `25`, как в оглавлении тома);
  - `みんな初級II-第NN課` — урок `NN` тома 初級 II, аналогично.
  При необходимости рядом можно оставлять общий тег `みんなの日本語` (если он уже используется в колоде). Если слово **не входит** в программу Minna (только расширенная лексика, другой источник и т.д.), тег урока **не добавляют**. Если в учебнике формально затронуто несколько уроков, берут **урок первичного введения**; второй тег того же формата — только при явной педагогической необходимости.

## Правила для картинки

- Картинка создаётся по смыслу третьего примера из поля **Пример перевод** и должна акцентировать внимание на изучаемом слове.
- Изучаемое слово должно быть визуальным центром сцены: главный объект, главное действие или самый заметный смысловой фокус иллюстрации.
- Финальная картинка должна быть полноценной raster AI-иллюстрацией или другой качественной готовой иллюстрацией по смыслу карточки, а не примитивной программной заглушкой.
- Локальный файл сохраняется в `Картинки/<тема>/` с тем же базовым именем, что и JSON-карточка: `<Слово>.jpg` или `<Слово>.png`.
- Папка `<тема>` должна совпадать с темой исходного списка в `Слова/` и JSON-файла в `Карточки/`.
- Пока изображение ещё не сгенерировано, поле **Картинка** в JSON считается источником промпта.
- Если есть техническая возможность создать изображение, картинку нужно сгенерировать по этому промпту сразу, без дополнительных вопросов пользователю, сохранить локально в `Картинки/<тема>/` и использовать этот файл для заполнения поля **Картинка** в Anki.
- После загрузки изображения в Anki поле **Картинка** в самой заметке Anki должно содержать картинку, например через HTML или ссылку на импортированный файл.
- После обратной синхронизации JSON поле **Картинка** может быть заменено на HTML или ссылку из Anki. Это нормально для уже обработанной карточки.
- Если изображение нужно сгенерировать заново, сначала в JSON поле **Картинка** нужно снова записать новый текстовый промпт по смыслу третьего примера, а уже потом повторять цикл генерации и синхронизации.
