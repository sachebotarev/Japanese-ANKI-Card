# Озвучка

Правила этого раздела применяются только к карточкам, у которых в JSON поле **Произношение** пустое.

## Общие правила

- Озвучка генерируется локально через `AivisSpeech`.
- Для генерации используется скрипт `scripts/generate_aivis_audio.py`.
- По умолчанию голос для озвучки выбирается случайно из доступных голосов локального `AivisSpeech`, которые поддерживают нужный стиль.
- По умолчанию скорость озвучки в `AivisSpeech` должна быть `90%` (`speedScale = 0.9`).
- Если нужен конкретный голос, сначала нужно проверить доступные стили через API `AivisSpeech` и затем указать его явно.

## Что нужно создать

Для каждой карточки должны существовать ровно четыре MP3-файла:

- `<Слово>_слово.mp3`
- `<Слово>_пример_1.mp3`
- `<Слово>_пример_2.mp3`
- `<Слово>_пример_3.mp3`

Все четыре файла сохраняются в `Произношение/<тема>/`, сохраняя структуру проекта.

## Что озвучивается

- отдельный файл для значения поля **Слово**;
- отдельный файл для первого примера из поля **Пример** без HTML-разметки;
- отдельный файл для второго примера из поля **Пример** без HTML-разметки;
- отдельный файл для третьего примера из поля **Пример** без HTML-разметки.

## Обновление JSON и Anki

- Поле **Произношение** в JSON обновляется через `--write-json` или вручную в формате `[sound:<Слово>_слово.mp3][sound:<Слово>_пример_1.mp3][sound:<Слово>_пример_2.mp3][sound:<Слово>_пример_3.mp3]`.
- Если поле **Произношение** в Anki ещё пустое, в него нужно добавить ссылки на созданные аудиофайлы.
- Если поле **Произношение** в Anki уже заполнено, его не нужно перезаписывать только потому, что в JSON оно ещё пустое.

## Базовый запуск

```bash
python3 scripts/generate_aivis_audio.py --write-json 'Карточки/Глаголы/乾かす.json'
python3 scripts/generate_aivis_audio.py --all-empty --write-json
```

## Поведение скрипта

- использует локальный `AivisSpeech Engine` на `http://127.0.0.1:10101`;
- если engine не запущен, пытается стартовать его из установленного `AivisSpeech.app`;
- по умолчанию выставляет `speedScale = 0.9`;
- ожидает, что в поле `Пример` лежат ровно 3 непустых примера нового формата, разделённых тегом `<br>`;
- сохраняет аудио в `Произношение/<тема>/`;
- обновляет поле `Произношение` в JSON только при флаге `--write-json`.
